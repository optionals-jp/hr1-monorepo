# HR1 戦略的機能計画

## Context

HR1は採用（ATS）+ 360度評価 + 社内ポータルを統合した中小企業向けHR SaaSプラットフォーム。3アプリ（Console/Employee/Applicant）+ Supabase バックエンドで構成され、機能実装は85-92%完了している。しかし**事業として成立するための基盤（課金・セルフサインアップ・メール通知）が未実装**であり、データ整合性の致命的欠陥も3件残存している。本計画はMVP1リリース→GA到達までの戦略的ロードマップを定義する。

ターゲット: 20〜80名のサービス業・IT/クリエイティブ・医療介護
ポジショニング: SmartHR（600〜1500円/人）の半額以下でATS+360度評価+ポータルを提供

## 1. 追加すべき機能（New Features）

### P0: 事業成立に必須（リリース前ブロッカー）

| # | 機能 | 概要 | 根拠 | 対象 | 工数 |
|---|------|------|------|------|------|
| 1-1 | **Stripe課金 + セルフサインアップ** | Consoleからアカウント作成→Stripe Checkoutでプラン選択→org自動作成。Webhookでsubscription管理 | 課金手段なしでは事業不成立。全収益の入口 | Backend（`subscriptions`/`billing_events`テーブル、Webhook Edge Function）、Console（`/signup`, `/settings/billing`） | XL |
| 1-2 | **CSV社員一括インポート** | テンプレートDL→アップロード→バリデーション→プレビュー→確定。create-user Edge Functionをバッチ呼び出し | 50人手入力で90%脱落。オンボーディング成功率に直結 | Console（`/employees`にインポートダイアログ）、Backend | M |
| 1-3 | **メール通知エンジン** | Resend/SendGrid連携Edge Function。承認依頼・面接確定・お知らせの3テンプレート。`notification_preferences`テーブルでチャネル切替 | プッシュのみではアプリ未インストール社員に届かない。承認フローが停止する | Backend（Edge Function）、Console（設定画面）、Employee App（通知設定） | M |

## フェーズ別ロードマップ

### Alpha（4週間）→ MVP1リリース
- 施策 1〜6: P0データ修正、Stripe課金、CSV、メール、テスト基盤、LP
- **ゴール**: 有料顧客を受け入れ可能な状態

### Closed Beta（8週間）→ 5-10社
- 施策 7〜11: UI修正群、RBAC、オンボーディング、Kanban、遷移UX
- **ゴール**: 実運用フィードバック収集、チャーン率測定

### Open Beta（8週間）→ 拡大
- 施策 12〜15 + 2-9, 2-11: 通知統合、検索、ポータル、メッセージ強化、ページネーション
- **ゴール**: セルフサインアップ公開、月次ARR成長

### GA → 有料プラン正式運用
- Starter 300円/人/月、Standard 500円/人/月、Premium 800円/人/月
- P2/P3施策の優先度を顧客フィードバックに基づき再評価

## 重要ファイル

- `hr1-console/src/middleware.ts` — RBAC実装のエントリポイント
- `hr1-console/src/types/database.ts` — 全型定義の中心（新テーブル型追加の起点）
- `supabase/functions/create-user/index.ts` — CSV一括インポートで流用するEdge Function
- `hr1-console/src/app/workflows/page.tsx` — 承認通知欠落（2-4）の修正対象
- `hr1-employee-app/lib/features/auth/presentation/screens/profile_edit_screen.dart` — P0修正（2-3）の対象
- `supabase/migrations/` — DBトリガー追加（2-1, 2-2）の配置先

## 検証方法

1. **P0データ整合性**: マイグレーション適用後、ワークフロー承認→leave_balancesの値変化をSQLで確認
2. **Stripe課金**: テストモードでサインアップ→プラン選択→Webhookイベント受信→org作成を確認
3. **CSV一括インポート**: 50行CSVをアップロード→全社員がprofiles/user_organizationsに作成されることを確認
4. **メール通知**: Resendテスト環境で承認依頼メールの送信・受信を確認
5. **テスト基盤**: `npm run test`（Console）/`flutter test`（Apps）でテスト実行、CI通過を確認
6. **全体**: `npm run format && npm run lint && npm run build && npm run test`（Console）
