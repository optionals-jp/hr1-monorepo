# HR1 採用管理機能 — 競合分析 & 戦略的ブラッシュアップ計画

## Context

HR1は日本市場向けのHR SaaSプロダクトで、採用管理（ATS）機能を中心に、Console（管理側）とApplicant App（応募者側）を提供している。現在、基本的なATS機能（求人管理、選考ワークフロー、面接日程調整、カスタムフォーム、メッセージング、ダッシュボード）は実装済みだが、HRMOS・HERP Hire・talentio等の競合と比較すると複数の領域で機能の深さが不足している。

**方針**: 全く新しい機能カテゴリを追加するのではなく、既存機能をブラッシュアップし、競合水準まで引き上げる。

## 1. 競合ギャップ分析

### 1.1 主要競合の強み

| 競合 | 最大の強み | HR1との差分 |
|------|-----------|------------|
| **HRMOS採用** | データ分析・レポート（応募経路別通過率、採用コスト、面接官評価分析）、AI書類マッチング、求人原稿AI生成 | HR1のダッシュボードは概要レベル。経路別分析・コスト分析・面接官分析なし |
| **HERP Hire** | スクラム採用（Slack上で評価・コメント完結）、30+求人媒体自動連携、Googleカレンダー空き枠抽出→日程調整URL自動発行 | HR1はSlack連携なし、求人媒体連携なし、カレンダー連携なし |
| **talentio** | タレントプール、リファラル採用、採用ページビルダー（多言語対応）、Marketo連携 | HR1はタレントプール・リファラル機能なし、採用ページビルダーなし |
| **ジョブカン採用管理** | LINE連携（管理画面内送受信）、低コスト・シンプルUI、アルバイト採用対応 | HR1はLINE連携なし |
| **sonar ATS** | 選考フロー図の可視化、エージェント管理、自動リマインド | HR1はエージェント管理なし、自動リマインド機能が弱い |
| **採用一括かんりくん** | LINE属性別コンテンツ配信、定型業務70%自動化、継続率98.8% | HR1は自動化・LINE配信なし |

### 1.2 機能別ギャップマトリクス

| 機能領域 | HR1現状 | 競合水準 | ギャップ |
|---------|---------|---------|---------|
| **求人作成・公開** | 基本情報のみ（タイトル、説明、部門等） | AI求人原稿生成、採用ページビルダー、SEO対応、多言語 | 🔴 大 |
| **応募経路管理** | なし（手動登録のみ） | 30+媒体自動連携、経路別分析、エージェント管理 | 🔴 大 |

| **日程調整** | 手動スロット作成→応募者選択 | カレンダー連携、空き枠自動抽出、日程調整URL、Zoom URL自動発行 | 🟡 中 |
| **分析・レポート** | ダッシュボード概要（KPI、パイプライン、目標進捗） | 応募経路別通過率、採用コスト、面接官評価、ファネル分析 | 🟡 中 |
| **コミュニケーション** | アプリ内メッセージング（リアルタイム） | メールテンプレート、LINE連携、Slack通知、自動リマインド | 🟡 中 |
| **チーム協業** | 基本的な権限管理 | Slack上で評価完結、@メンション、面接官アサイン、評価リマインド | 🟡 中 |
| **選考ワークフロー** | カスタムステップ（書類/フォーム/面接/外部テスト/内定） | フロー図可視化、条件分岐、自動進行 | 🟢 小 |
| **面接管理** | 面接作成・スロット管理・確定 | ビデオ面接連携、録画、AI文字起こし | 🟢 小 |
| **カスタムフォーム** | 多様なフィールドタイプ対応済み | ほぼ同等 | ✅ 充足 |
| **応募者→社員変換** | 実装済み | ほぼ同等（HRMOS COREのような連携が差分） | ✅ 充足 |
| **モバイルアプリ** | Flutter製の本格アプリ（メッセージ、TODO、面接調整等） | 多くの競合はWebのみ | ✅ **優位** |
| **リアルタイム更新** | Supabase Realtime実装済み | 多くの競合は未対応 | ✅ **優位** |

## 2. 優先改善領域（既存機能ブラッシュアップ）

既存機能の延長線上にある改善を、**インパクト × 実装コスト** で優先順位付け。

### Priority 1: 選考評価の構造化（高インパクト・中コスト）
**現状**: 選考ステップは「ステータス変更」のみ。評価理由や面接評価が記録されない。  
**目標**: 面接官が構造化された評価を入力でき、候補者ごとに評価が集約される。

- `evaluation_templates`, `evaluations`, `evaluation_scores` テーブルは**既にDB上に存在**
- Console側に評価入力UI・評価サマリー表示を追加
- 面接ステップと評価テンプレートの紐づけ機能
- 面接官別の評価傾向レポート

**対象ファイル**:
- `hr1-console/src/features/applications/components/application-steps-tab.tsx` — 評価入力UIを追加
- `hr1-console/src/features/applications/hooks/use-application-detail.ts` — 評価データ取得ロジック
- `hr1-console/src/lib/repositories/application-repository.ts` — 評価CRUD追加
- `hr1-console/src/types/database.ts` — 評価関連型は既存

### Priority 2: 分析・レポート強化（高インパクト・中コスト）
**現状**: ダッシュボードは概要KPIのみ。経路別・コスト別の深掘り分析がない。  
**目標**: HRMOS水準の分析機能。

- 応募経路トラッキング（`applications`テーブルに`source`カラム追加）
- 選考ファネル分析（ステップ別通過率・離脱率）
- 採用リードタイム分析（応募→内定の所要日数）
- 面接官別通過率レポート（評価機能と連動）
- 部門別・求人別の詳細レポートページ新設

**対象ファイル**:
- `hr1-console/src/lib/dashboard/data-sources.ts` — 新データソース追加
- `hr1-console/src/components/dashboard/` — 新チャートコンポーネント
- `hr1-console/src/app/(dashboard)/reports/` — レポート専用ページ（新規）

### Priority 3: 日程調整の高度化（中インパクト・中コスト）
**現状**: 手動でスロット作成→応募者がアプリで選択。  
**目標**: カレンダー連携による空き枠自動提案。

- Googleカレンダー連携（面接官の空き枠取得）
- 日程調整URL発行（外部ツール不要で候補者に共有）
- Web会議URL自動発行（Zoom/Google Meet連携）
- 面接リマインド自動送信（面接24時間前・1時間前）

**対象ファイル**:
- `hr1-console/src/lib/repositories/scheduling-repository.ts` — カレンダーAPI連携
- `hr1-console/src/app/(dashboard)/scheduling/` — UI改善
- `supabase/functions/` — リマインド用Edge Function追加

### Priority 4: コミュニケーション自動化（中インパクト・小コスト）
**現状**: アプリ内メッセージは実装済みだが、テンプレートや自動送信がない。  
**目標**: 定型コミュニケーションの効率化。

- メッセージテンプレート機能（選考通過/不通過/面接案内等）
- ステータス変更時の自動メッセージ送信（opt-in）
- 一括メッセージ送信（同一ステータスの応募者へ）
- メールテンプレート（アプリ外の候補者向け）

**対象ファイル**:
- `hr1-console/src/features/applications/components/` — テンプレート選択UI
- `hr1-console/src/lib/repositories/` — テンプレートCRUD
- DB: `message_templates`テーブル追加

### Priority 5: 求人ページの充実（中インパクト・小コスト）
**現状**: 求人は管理画面上での閲覧・編集のみ。外部公開用ページなし。  
**目標**: 自社採用ページとしても機能する求人公開機能。

- 求人詳細のリッチ化（セクション構造、画像、福利厚生等）
- 公開用求人ページURL発行（OGP対応）
- 応募フォーム埋め込み（外部サイトへのiframe）
- 求人一覧の公開ページ（企業採用サイト代替）

**対象ファイル**:
- `hr1-console/src/app/(dashboard)/jobs/` — 求人編集UI拡張
- 新規: 公開用求人ページ（別ルート or サブドメイン）

### Priority 6: Slack/外部ツール連携（中インパクト・大コスト）
**現状**: 外部ツール連携なし。  
**目標**: HERP Hire水準のSlack連携。

- Slack通知（新規応募、面接リマインド、評価依頼）
- Slack上での評価入力（Slackアクション）
- Webhook対応（外部システムへのイベント通知）

## 3. チーム構成提案

### 3.1 推奨チーム（6名体制）

| 役割 | 人数 | 主な責務 |
|------|------|---------|
| **プロダクトマネージャー (PM)** | 1名 | 競合分析の継続、優先順位判断、ユーザーヒアリング、ロードマップ管理 |
| **フロントエンドエンジニア (Console)** | 2名 | Next.js Console側の評価UI、レポートページ、日程調整改善、テンプレート機能 |
| **バックエンドエンジニア (Supabase)** | 1名 | DBマイグレーション、RPC関数、Edge Functions、外部API連携（Calendar/Zoom） |
| **モバイルエンジニア (Flutter)** | 1名 | Applicant App側の評価閲覧、リマインド通知、求人ページ表示改善 |
| **デザイナー (UI/UX)** | 1名 | 評価フォームUI、レポート画面、日程調整フロー、求人ページテンプレート |

### 3.2 役割分担イメージ

```
PM ─── 競合ウォッチ・ユーザーインタビュー・優先順位調整
 │
 ├── FEチーム（2名）
 │   ├── FE-1: 評価機能 → レポート画面 → Slack連携
 │   └── FE-2: 日程調整改善 → テンプレート → 求人ページ
 │
 ├── BEエンジニア（1名）
 │   └── DB設計 → RPC → Edge Functions → 外部API
 │
 ├── モバイル（1名）
 │   └── 評価閲覧 → 通知改善 → 求人ページ
 │
 └── デザイナー（1名）
     └── 全機能のUI/UXデザイン（1スプリント先行）
```

## 4. フェーズ別ロードマップ

### Phase 1（Q2 2026 — 3ヶ月）: 選考の質を上げる
**テーマ**: 評価の構造化 + 分析基盤

| 施策 | 担当 | 詳細 |
|------|------|------|
| 選考評価フォーム | FE-1 + BE + Design | 既存の`evaluation_templates`/`evaluations`/`evaluation_scores`テーブルを活用。Console上に評価入力UI・サマリー表示を実装 |
| 応募経路トラッキング | BE + FE-2 | `applications.source`カラム追加、手動入力 + インポート時の経路記録 |
| 選考ファネルレポート | FE-1 + Design | ステップ別通過率・離脱率のファネルチャート。`data-sources.ts`に新データソース追加 |
| 採用リードタイム分析 | FE-2 | 応募→内定の所要日数分布。既存の`applied_at`と`completed_at`を活用 |
| モバイル: 評価結果閲覧 | Mobile | 応募者が自分の選考進捗をより詳細に確認できるよう改善 |

**検証方法**:
- `npm run build` & `npm run test` が通ること
- 評価テンプレート作成→面接評価入力→評価サマリー表示の一連のフロー確認
- ファネルレポートが正しいデータを表示すること

### Phase 2（Q3 2026 — 3ヶ月）: 業務効率を上げる
**テーマ**: 日程調整高度化 + コミュニケーション自動化

| 施策 | 担当 | 詳細 |
|------|------|------|
| Googleカレンダー連携 | BE + FE-2 | OAuth2で面接官のカレンダー接続、空き枠自動抽出 |
| 日程調整URL発行 | BE + FE-2 | 候補者にURLを送り、空き枠から選択させるフロー |
| Web会議URL自動発行 | BE | Zoom/Google Meet API連携でURL自動生成 |
| メッセージテンプレート | FE-1 + Design | テンプレートCRUD + 送信時テンプレート選択 |
| 自動リマインド | BE | 面接前リマインド（24h/1h前）のEdge Function |
| ステータス変更時の自動メッセージ | FE-1 + BE | opt-in設定で選考通過時等に自動メッセージ送信 |
| モバイル: 通知・リマインド強化 | Mobile | プッシュ通知の種類拡充、面接前リマインド |

**検証方法**:
- カレンダー連携→空き枠表示→候補者選択→確定の一連フロー
- テンプレート作成→テンプレートからメッセージ送信
- 面接前リマインドが時間通りに送信されること

### Phase 3（Q4 2026 — 3ヶ月）: 候補者体験を上げる
**テーマ**: 求人ページ充実 + 面接官別レポート

| 施策 | 担当 | 詳細 |
|------|------|------|
| 求人ページリッチ化 | FE-2 + Design | セクション構造（仕事内容、応募要件、福利厚生、チーム紹介等） |
| 公開求人ページ | FE-2 + BE | 外部公開URL、OGP対応、応募フォーム |
| 面接官別レポート | FE-1 | 面接官ごとの通過率・評価傾向分析（Phase 1の評価データ活用） |
| 応募経路別ROIレポート | FE-1 | 経路別の採用コスト・通過率・内定率 |
| 一括メッセージ送信 | FE-1 + BE | 同一ステータスの応募者への一括連絡 |
| モバイル: 求人ページ表示改善 | Mobile | リッチ化された求人詳細の表示対応 |

**検証方法**:
- 公開求人ページがURLでアクセスでき、OGPが正しく表示されること
- 面接官別レポートがダミーデータで正しく集計されること

### Phase 4（Q1 2027 — 3ヶ月）: 外部連携で生産性を上げる
**テーマ**: Slack連携 + API/Webhook

| 施策 | 担当 | 詳細 |
|------|------|------|
| Slack通知連携 | BE + FE-1 | 新規応募、面接リマインド、評価依頼のSlack通知 |
| Slack評価入力 | BE | Slackアクション経由での簡易評価入力 |
| Webhook/API公開 | BE | 採用イベントの外部通知、サードパーティ連携基盤 |
| 求人媒体連携（主要3社） | BE + FE-2 | Wantedly、Green、ビズリーチからの応募自動取り込み |

## 5. HR1の競合優位性（維持・強化すべきポイント）

| 強み | 説明 |
|------|------|
| **ネイティブモバイルアプリ** | 多くの競合はWebベースのみ。Flutter製アプリは候補者体験で大きな差別化 |
| **リアルタイム機能** | Supabase Realtimeによるメッセージング・ステータス更新は競合優位 |
| **柔軟な選考ワークフロー** | カスタムステップ対応は十分な水準 |
| **応募者→社員変換** | 採用から入社までのシームレスな移行は一体型HR SaaSならではの強み |
| **TODO/タスク管理** | 応募者側のタスク管理は競合にない独自機能 |
