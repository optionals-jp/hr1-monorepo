# HR1 契約フロー

## 概要

企業が HR1 を契約し、利用開始するまでのフローを定義する。

---

## 現状のフロー（手動オペレーション）

セルフサインアップは未実装のため、全ステップが手動。

```
┌─────────────────────────────────────────────────────────────┐
│ Step 1: 企業情報の登録                                        │
│   HR1運営者が Supabase Studio で organizations に INSERT       │
│   └─ id (text), name, industry, location 等                  │
├─────────────────────────────────────────────────────────────┤
│ Step 2: 契約の作成                                            │
│   HR1運営者が hr1-admin → 契約管理 → 新規契約                   │
│   └─ 企業選択, プラン選択, 社員数, 月額, 開始日                  │
├─────────────────────────────────────────────────────────────┤
│ Step 3: 管理者アカウントの作成                                  │
│   HR1運営者が Supabase Dashboard → Authentication → Users     │
│   └─ メール + パスワードでユーザー作成                           │
│   └─ profiles テーブルに role='admin' で INSERT                │
│   └─ user_organizations に organization_id を紐付け            │
├─────────────────────────────────────────────────────────────┤
│ Step 4: 管理者がコンソールにログイン                              │
│   企業管理者が hr1-console にメール+パスワードでログイン            │
├─────────────────────────────────────────────────────────────┤
│ Step 5: 社員の登録                                            │
│   企業管理者がコンソール → 社員管理 → 社員追加                    │
│   └─ create-user Edge Function で社員アカウント + profile 作成  │
│   └─ OTP メールが社員に送信される                                │
├─────────────────────────────────────────────────────────────┤
│ Step 6: 社員がアプリにログイン                                   │
│   社員が hr1-employee-app でメールアドレス入力 → OTP認証          │
│   └─ 組織は自動的に紐付き済み                                   │
├─────────────────────────────────────────────────────────────┤
│ Step 7: 求人の作成・応募者の利用                                 │
│   企業管理者がコンソール → 求人管理 → 求人作成                    │
│   応募者が hr1-applicant-app で OTP ログイン → 組織選択 → 応募    │
└─────────────────────────────────────────────────────────────┘
```

---

## 各ステップの実装状況

| Step | 内容 | 実装状況 | 担当 |
|------|------|---------|------|
| 1 | 企業登録 | 未実装（Supabase Studio で手動） | hr1-admin |
| 2 | 契約作成 | 実装済み | hr1-admin |
| 3 | 管理者アカウント作成 | 未実装（Supabase Dashboard で手動） | hr1-admin |
| 4 | コンソールログイン | 実装済み | hr1-console |
| 5 | 社員登録 | 実装済み（create-user Edge Function） | hr1-console |
| 6 | 社員アプリログイン | 実装済み（OTP） | hr1-employee-app |
| 7 | 求人作成・応募 | 実装済み | hr1-console / hr1-applicant-app |

---

## 未実装の課題

### 必須（MVP1）

| # | 課題 | 説明 |
|---|------|------|
| 1 | **hr1-admin に企業作成 UI がない** | organizations ページは閲覧のみ。新規企業を作成するフォームが必要 |
| 2 | **hr1-admin に管理者アカウント作成 UI がない** | 契約作成時に初期管理者のメールアドレスを入力し、自動でアカウント作成+招待メール送信する機能が必要 |
| 3 | **セルフサインアップ未実装** | 企業が自分で HR1 に登録できない。LP → サインアップ → 組織作成 → トライアル開始のフローが必要 |
| 4 | **Stripe 課金連携未実装** | トライアル終了後の課金手段がない |

### 理想のフロー（将来）

```
┌─────────────────────────────────────────────────────────────┐
│ 1. LP からサインアップ                                         │
│    企業管理者がメール + パスワードで登録                          │
├─────────────────────────────────────────────────────────────┤
│ 2. 組織情報入力                                               │
│    企業名, 業種, 規模 → organizations + contracts 自動作成      │
│    └─ 30日トライアル開始（Stripe Customer 作成）                │
├─────────────────────────────────────────────────────────────┤
│ 3. コンソールに自動遷移                                        │
│    オンボーディングウィザード（社員 CSV インポート等）              │
├─────────────────────────────────────────────────────────────┤
│ 4. トライアル終了前に課金設定                                    │
│    Stripe Checkout でカード登録 → 自動課金開始                   │
└─────────────────────────────────────────────────────────────┘
```
