hr1-console（Next.js管理コンソール）のコードレビューを実施してください。

## 手順

1. `git diff HEAD -- hr1-console/` で hr1-console 配下の変更を確認する
2. 変更されたファイルを全て読み込む
3. 以下の観点でレビューする

## レビュー観点

### 必須チェック
- **バグ・ロジックエラー**: off-by-one、null参照、型不一致、条件分岐の漏れ
- **セキュリティ**: XSS、SQLインジェクション、認証/認可の漏れ、機密情報のハードコード、RLSポリシーの整合性
- **データ整合性**: DB操作の整合性、外部キー制約、トランザクション境界、ロールバック処理

### コード品質
- **命名**: 変数・関数名が処理内容を適切に表しているか
- **重複**: 同じロジックがコピペされていないか
- **未使用コード**: importやvariableが使われずに残っていないか
- **エラーハンドリング**: 適切にエラーが処理されているか（過剰なハンドリングは不要）

### Next.js / React 固有
- **"use client" / "use server"**: クライアント・サーバーコンポーネントの分離が適切か
- **React Hooks**: useEffect の依存配列、useMemo/useCallback の使い方
- **状態管理**: 不要な再レンダリングを引き起こしていないか

### プロジェクト固有
- **UIテキスト**: 日本語が適切か
- **Supabaseクライアント**: `@supabase/ssr` 経由（`createBrowserClient` / `createServerClient`）で使用しているか。`@supabase/supabase-js` の直接使用は禁止
- **型定義**: `src/types/database.ts` との整合性
- **Tailwind CSS**: `w-[Npx]` のような任意値クラスではなく、標準クラス（w-10, w-20 等）を優先しているか

### ビルド・Lint チェック
- レビュー完了後、`cd hr1-console && npm run format:check && npm run lint && npm run build` を実行し、エラーがないか確認する

## 出力フォーマット

レビュー結果を以下の形式で報告してください：

### 🔴 要修正（マージ前に修正が必要）
### 🟡 推奨（修正を推奨するが必須ではない）
### 🟢 良い点（特に良い実装）

各項目には **ファイル名:行番号** と **理由** を明記してください。

問題がない場合は「レビュー完了：問題なし ✅」と報告してください。
