hr1-admin（Next.js HR1運営管理画面）のコードレビューを実施してください。

## 概要

hr1-admin は HR1 サービス運営者向けの管理画面です。契約企業管理、プラン管理、MRR ダッシュボード等を提供します。
hr1-console（テナント企業向け管理コンソール）とは別のアプリです。

## 手順

1. `git diff HEAD -- hr1-admin/` と `git diff origin/main...HEAD -- hr1-admin/` で変更を確認する
2. 変更がある場合は変更されたファイルを全て読み込む。変更がない場合は `hr1-admin/src/` 配下の全ページ・コンポーネント・hooks を対象にアプリ全体レビューを実施する
3. 以下の観点でレビューする

## レビュー観点

### 必須チェック
- **バグ・ロジックエラー**: off-by-one、null参照、型不一致、条件分岐の漏れ
- **セキュリティ**: 運営管理画面のため認証/認可が特に重要。一般ユーザーやテナント管理者がアクセスできないこと。XSS、SQLインジェクション、機密情報のハードコード
- **データ整合性**: 契約・プラン・課金データの整合性。誤操作防止（確認ダイアログ等）

### コード品質
- **命名**: 変数・関数名が処理内容を適切に表しているか
- **重複**: 同じロジックがコピペされていないか
- **未使用コード**: importやvariableが使われずに残っていないか
- **エラーハンドリング**: 適切にエラーが処理されているか（過剰なハンドリングは不要）

### Next.js / React 固有
- **"use client" / "use server"**: クライアント・サーバーコンポーネントの分離が適切か
- **React Hooks**: useEffect の依存配列、SWR の使い方
- **状態管理**: 不要な再レンダリングを引き起こしていないか

### Supabase クライアント
- **SSR クライアント**: `@supabase/ssr` 経由で使用しているか
- **権限**: 運営管理者のみがアクセスできるテーブル・RPC のアクセス制御
- **型定義**: `src/types/database.ts` との整合性

### UI / スタイリング
- **UIテキスト**: 日本語が適切か（運営者向けの表現になっているか）
- **shadcn/ui**: 共通コンポーネント（Card, Table, Badge, Dialog 等）を適切に使用しているか
- **Tailwind CSS**: 任意値クラスではなく標準クラスを優先しているか

### 運営管理固有
- **ダッシュボード**: KPI（MRR、契約数、社員数）の計算が正しいか
- **契約管理**: ステータス遷移（trial → active → suspended → cancelled）が正しいか
- **プラン管理**: 料金計算の整合性
- **企業管理**: テナントデータの参照が正しいか（cross-tenant でのデータ漏洩がないか）

### ビルド・Lint チェック
- レビュー完了後、`cd hr1-admin && npm run format:check && npm run lint && npm run test && npm run build` を実行し、エラーがないか確認する

## 出力フォーマット

レビュー結果を以下の形式で報告してください：

### 🔴 要修正（マージ前に修正が必要）
### 🟡 推奨（修正を推奨するが必須ではない）
### 🟢 良い点（特に良い実装）

各項目には **ファイル名:行番号** と **理由** を明記してください。

問題がない場合は「レビュー完了：問題なし ✅」と報告してください。
